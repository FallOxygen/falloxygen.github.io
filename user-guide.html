📚 准备曲库

网站启动后默认进入"曲库"选项卡。你可以选择导入内置的在线曲库，或者按照指定格式导入自己的本地曲库。

1.1 导入在线曲库

点击橙色的 导入在线曲库 按钮，系统会自动从 GitHub 拉取最新的内置曲库文件。导入成功后，下方会显示曲库包含的游戏范围数量和总曲目数。

网络要求
在线导入需要科学上网环境，若导入失败请检查网络连接，或改用本地导入。

1.2 导入本地曲库

点击 导入本地 按钮，在弹窗中输入曲库名称（可选）、允许的范围标签（必须，一行一个），并选择本地的 .txt 曲库文件。

TXT 文件格式要求：

  • 一行 [游戏名] 作为范围声明。
  • 下面逐行写曲名，可以不带序号，系统会自动去除前缀序号（如 1.、2) 等）。
[Phigros]
1.Distorted Fate
2.DESTRUCTION 3,2,1
3.Glaciaxion
[Arcaea]
1.Sheriruth
2.Arcahv
范围标签冲突
如果你自定义的范围标签与内置标签同名（如也叫 Phigros），会导致数据冲突。导入本地时请务必声明该曲库包含哪些标签，系统只会识别声明的标签。

1.3 曲库管理与编辑

在曲库下拉框旁，有 重命名 和 删除 按钮（内置在线曲库不可重命名/删除，只能清空内容）。

直接编辑曲库：在游戏范围列表中，每个范围右侧有一个 ✎ 按钮。点击后会弹出编辑窗口，你可以：

  • 搜索：在顶部搜索框过滤曲目，快速定位目标。
  • 删除：点击曲目右侧的 X 删除该曲目。
  • 添加：在底部输入新曲名并点击"添加"，也可以直接按 Enter 键快速添加。
  • 保存/重置：点击"保存修改"会将变更持久化存入浏览器本地（IndexedDB）；点击"重置回初始状态"会撤销未保存的修改，恢复到导入时的原始数据。
数据持久化
曲库编辑数据保存在浏览器的 IndexedDB 中，清除浏览器数据会导致本地曲库丢失，建议保留原始 TXT 文件备份。

1.4 曲库导出

在设置面板底部新增了 曲库导出 功能。你可以：

  • 在下拉框中选择要导出的游戏范围。
  • 勾选"仅导出只有这个游戏收录的曲目"，可过滤掉与其他范围共享的曲目，只保留独占曲目。
  • 点击"导出为 TXT"后，系统会生成标准格式的曲库文件并自动下载。

此功能方便你将编辑后的曲库或某个范围的独占曲目备份为本地文件。

⚙️ 配置出题序列

2.1 选择范围与过滤

导入曲库后，在"选择游戏范围"卡片中勾选你要参与抽取的游戏（如 Phigros, Arcaea）。

曲目过滤功能：你可以对选中的范围进行进一步筛选：

  • 字符类型：限制曲名必须包含（或不包含）字母、日文、汉字。未勾选的字符类型将被排除。
  • 允许其他字符：勾选后，曲名中可以包含标点、数字等非字母/日文/汉字字符；不勾选则仅允许已选字符类型。
  • 最少字符长度：过滤掉名字太短的歌（计算时包含空格）。设为 0 表示不限。
  • 排除字符：曲名中不能包含你输入的任何字符。
  • 包含字符：曲名必须包含你输入的字符。可勾选"必须同时包含所有输入的字符"来启用严格模式。
  • 仅抽取独占曲目：开启后，只会抽取"仅出现在已选范围中"的曲目。如果某首歌同时出现在已选范围和未选范围中，则会被排除。适合制作"独占猜歌"挑战。

过滤条件会实时显示"符合条件的曲目"数量，方便你调整参数。

2.2 抽取设置

系统提供两种抽取模式：

  • 总数量模式：输入一个总数，系统会从你选中的所有范围里均匀、随机地抽取指定数量的曲目，避免同一范围过度集中。
  • 定制数量模式：为每个选中的游戏范围分别指定要抽取的数量，精确控制各范围占比。

配置好后点击 生成曲目列表，系统会自动跳转到"手动输入"选项卡，并填入生成的曲目。

2.3 手动输入与正则选曲

你也可以直接在"手动输入"选项卡的文本框中粘贴曲名（每行一首，无需序号）。

从曲库手动选曲：在"曲库"选项卡中点击 从曲库手动选曲...。

  • 支持正则表达式搜索，例如输入 ^A.* 可以搜索所有以 A 开头的歌，输入 Phigros 可以按范围筛选。
  • 点击曲目行即可选中/取消选中，选完后点击"导入到出题序列"。
  • 最多显示 300 条搜索结果，如果结果过多请缩小搜索范围。

排列选项：你可以将当前文本框中的曲目进行"随机排列"或"按长度排列"。点击"应用排列"生效。

最后，可以在"备注信息"中填写如"范围：Phigros, Arcaea"等提示信息，点击 解析并开始游戏 正式进入游戏界面。

2.4 难度增强（显示变换）

在设置面板中，新增了"难度增强（显示变换）"板块。在开始游戏前，你可以对曲名进行多种变换处理，大幅提升猜歌难度：

  • 空格隐藏为遮罩符号：开启后，曲名中的空格也会显示为遮罩符号（如 ?），玩家无法通过空格位置判断单词边界。
  • 英文字母大小写：可选择"不转换"、"全部小写"或"全部大写"。例如原曲名 Distorted Fate 在"全部小写"模式下会显示为 distorted fate。
  • 字符换位程度：0% 到 100% 的滑块。系统会随机交换曲名中字符的位置，百分比越高，打乱程度越大。例如 50% 可能会将 Hello 变为 eHllo。
  • 字母替换程度：0% 到 100% 的滑块。系统会随机将曲名中的英文字母替换为其他随机字母。例如 30% 可能会将 Arcahv 变为 Akcahx。
  • 最大拼合曲数：设为 1 时不拼合（默认）。设为大于 1 的数字后，系统会将同一范围的多个曲名随机拼合成一个长字符串，中间以空格分隔。例如将 Distorted Fate 和 Glaciaxion 拼合为 Distorted Fate Glaciaxion。拼合后的曲目在游戏中作为一个整体进行开字母和猜测。
变换顺序说明
系统实际执行的变换顺序为：先拼合 → 再换位 → 再替换 → 最后大小写转换。这意味着大小写转换会覆盖在其他变换之上。单独提示字符和查询来源功能始终基于变换后的显示文本操作，但查询来源会同时显示原始曲名。

🎮 游戏进行中

3.1 计时器与基础控制

页面顶部显示巨大的计时器，精确到秒。你可以点击 暂停 暂停计时，再次点击可继续。点击 结束游戏 进入结算页面。

如果上次有未完成的游戏，重新进入页面时会弹出恢复提示，可以选择"继续游戏"（恢复计时器和已开字符）或"开始新游戏"。

3.2 开启字符 (核心功能)

在"开启字符"输入框中，输入你想要公开的字母（如 aeiou）。输入后，下方所有曲目中对应的字母都会高亮显示（橙色），未开出的字符会变成遮罩符号（默认 ?）。

系统会自动处理以下等效规则：

  • 全角/半角等效：如开 ( 会同时开出 （，开 ! 会同时开出 ！ 等。
  • 忽略大小写：输入 a 会同时匹配 A。

下方有三个统计数据卡片：曲名数（剩余可见）、已开字符（你输入了几个字符）、揭示率（所有非空格字符中被开出的比例）。

3.3 各范围未出曲目统计

游戏界面新增了一个"各范围未出曲目统计"卡片，实时显示当前仍未被猜出的曲目按来源范围的分组统计。例如：

[Phigros] 3首
[Arcaea] 1首

点击"复制"按钮可将统计结果一键复制到剪贴板，方便主持人实时掌握各范围的剩余难度。

3.4 曲目交互操作

在"解析结果"列表中，每首歌右侧有两个小图标按钮：

  • 查询来源 (放大镜图标)：如果开启了"曲目来源查询"设置，点击此按钮可查看该曲目出现在哪些游戏范围中。如果曲目是拼合的，会分别查询每个原始曲目的来源。结果可以一键复制。
  • 单独提示 (太阳图标)：点击会弹出一个模态框，展示该曲目的所有字符（已应用显示变换后的字符）。你可以点击任意字符将其强制单独公开（显示为黄色高亮），这不会影响全局的已开字符计数和揭示率。适合作为特殊提示或主持人给线索。再次点击可取消单独提示。

显示全名与隐藏：直接点击曲目行，会显示完整的曲名（绿色高亮）。当曲目经过拼合变换后，显示的全名会是所有拼合的原始曲名，以 / 分隔。当玩家猜对后，你可以点击 隐藏已猜出 按钮，将所有处于"显示全名"状态的曲目从列表中移除（移入隐藏状态）。隐藏的数量会显示在按钮左侧。

3.5 撤销与重做

在"开启字符"输入框右侧，有 ↶ 撤销 和 ↷ 重做 按钮。每当你修改了开字符、隐藏了曲目或进行了单独提示操作，系统都会自动存入历史记录（最多保留 50 步）。你可以随时撤销误操作。

点击 重置 会清空所有开启的字符和单独提示，恢复到初始状态。

🏁 结算与导出

点击"结束游戏"后，会进入结算页面。页面包含三个主要部分：

  • 结算信息：包含用时、曲目总数、猜出数量（若全部猜出会显示"全部猜出"）、开启字符数、揭开率等详细数据。若开启了计时功能，还会显示平均用时（秒/首、秒/字符）。
  • 未猜出的曲目：列出玩家没能猜出来的歌曲全名及序号。如果全部猜出，会显示"太棒了，全部都猜出来了！"
  • 完整题面：本局游戏的所有曲目全名及备注，用于复盘和存档。

每个部分都有 复制 和 导出图片 按钮。导出图片功能会将文本生成为一张图片并直接下载保存。你可以通过"导出图片颜色设置"自定义图片的背景色和文字颜色，也可以开启"固定导出图片 9:16 比例"来生成适合手机分享的竖屏图片。

结算完毕后，可以选择"开始新游戏"或"返回编辑"。

🛠️ 全局设置详解

点击右上角的齿轮图标打开设置面板。所有设置会自动保存在浏览器本地。

遮罩符号

默认是 ?。你可以改成任意单个字符，如 * 或 _。

开字母限制

  • 最大开字符数：限制全局最多能开几个字符，防止直接把歌名开完。设为 0 为不限制。当玩家尝试输入超出限制的字符时，输入会被拦截并提示。
  • 最大揭开率限制：限制当前所有曲目被揭开的百分比上限。达到上限后无法继续开新字符。设为 0 为不限制。

功能开关

  • 开启计时功能：关闭后游戏界面将不显示计时器，结算时也不统计用时。
  • 开启结算功能：关闭后点击"结束游戏"会直接返回首页，不生成结算报告。
  • 空格也隐藏为遮罩符号：默认情况下空格正常显示，开启后空格也会变成遮罩符号，增加难度。
  • 自动根据游戏范围填充备注：生成曲目或开始游戏时，自动在备注栏写入"范围：XXX, YYY"。
  • 开启字符排序：将你输入的乱序字符按"数字 → 大写字母 → 小写字母 → 平假名 → 片假名 → 标点 → 汉字 → 其他"的顺序自动排序显示在顶部。
  • 开启曲目来源查询：在游戏界面每首歌旁显示放大镜按钮，用于查询来源。
  • 输出中显示开字母限制进度：导出/复制结果时，在顶部附加当前字符限制和揭开率的进度信息。
  • 输出中显示各范围未出曲目统计：导出/复制结果时，在顶部附加各范围剩余未猜出曲目的统计信息。
  • 固定导出图片 9:16 比例：导出图片时强制使用 1080×1920 的竖屏比例，适合手机分享。不勾选则按内容自适应宽高。

难度增强（显示变换）

这是 v17 版本新增的核心功能，用于在出题阶段对曲名进行多种变换，大幅提升猜歌难度。所有变换在点击"解析并开始游戏"时一次性应用，游戏中不可再修改。

  • 空格隐藏为遮罩符号：与功能开关中的同名选项效果相同，曲名中的空格显示为遮罩符号。
  • 英文字母大小写：可选择"不转换"（保持原样）、"全部小写"（如 Arcahv → arcahv）或"全部大写"（如 Arcahv → ARCAHV）。注意：大小写转换是最后一步执行的，会覆盖在其他变换之上。
  • 字符换位程度（0% ~ 100%）：随机交换曲名中字符的位置。0% 为不换位，100% 为完全打乱。例如 50% 的 Hello 可能变为 eHllo。
  • 字母替换程度（0% ~ 100%）：随机将曲名中的英文字母替换为其他随机字母。0% 为不替换，100% 为所有字母都被替换。例如 30% 的 Arcahv 可能变为 Akcahx。
  • 最大拼合曲数（1 为不拼合）：将同一来源范围的多个曲名随机拼合成一个长字符串显示。例如设为 3 时，可能将 2~3 首同范围的曲名拼在一起。拼合后的曲目作为一个整体进行开字母，点击显示全名时会展示所有拼合的原始曲名。
变换执行顺序
系统内部执行顺序为：拼合 → 换位 → 替换 → 大小写转换。这意味着如果你同时开启了多种变换，大小写转换会最后生效，且单独提示字符功能作用于最终变换后的文本。

导出图片颜色设置

你可以自定义导出图片的背景色和文字颜色，以适应不同的分享平台背景。修改后即时生效。

界面主题颜色

支持自定义整个网页界面的配色方案，包括：

  • 背景颜色：页面主背景色
  • 文字颜色：正文和标题颜色
  • 强调色：按钮、高亮、统计数字等强调元素的颜色
  • 卡片颜色：各功能卡片的面板背景色
  • 边框颜色：卡片边框、输入框边框等线条颜色

点击"重置为默认"可一键恢复原始暗色主题。

主题兼容性
主题颜色修改后会即时应用到整个页面。如果选择了对比度过低的颜色组合，可能会影响可读性，请谨慎调整。

曲库导出

在设置面板底部，你可以选择任意一个已导入的范围，将其曲目导出为 TXT 文件。支持勾选"仅导出只有这个游戏收录的曲目"来过滤独占曲目。导出的文件格式与导入格式一致，可直接用于备份或分享。

❓ 常见问题与联系

常见问题

Q: 数据会保存在哪里？

A: 游戏状态保存在浏览器的 localStorage 中，曲库数据保存在 IndexedDB 中。清除浏览器数据会导致丢失，建议保留原始曲库文件备份。

Q: 为什么有些字符开了没反应？

A: 请检查是否开启了"空格也隐藏为遮罩符号"，以及是否达到了"最大开字符数"或"最大揭开率"的限制。另外，如果曲名经过"字母替换"变换，原始字母可能已被替换，导致无法匹配。

Q: 独占曲目过滤是什么意思？

A: 开启后，只保留"仅出现在你勾选的范围中"的曲目。如果一首歌同时出现在已选范围和未选范围，则会被过滤掉。

Q: 拼合曲名后怎么算猜对？

A: 拼合后的曲目作为一个整体条目存在。玩家需要猜出所有拼合的曲名才算猜对这一"条"。点击曲目行显示全名时，会展示所有拼合的原始曲名，以 / 分隔。

Q: 导出图片为什么不显示预览？

A: 当前版本导出图片后直接触发下载，不再显示预览弹窗。如需调整图片样式，请先在设置中修改"导出图片颜色"。
